# CatBus — 协作文档库

> 架构文档、接口规格、各工程师工作区

## 目录结构

```
├── docs/           # 项目文档（架构 / API / 决策记录）
├── specs/          # 需求 & 规格（定稿）
├── workspace/      # 各工程师工作区
│   ├── nefi/       # 架构 & 协调（NeFi）
│   ├── lebron/     # 前端工程师
│   ├── rocs/       # 后端工程师
│   └── shared/     # 跨工程师共享
└── code/           # 代码片段 & 配置模板
```

## 工作规范

- `docs/` `specs/` 放**定稿内容**，改动需告知团队
- `workspace/<name>/` 是各自的草稿区，自由发挥
- `code/` 只放片段和模板，完整代码在各自 repo
