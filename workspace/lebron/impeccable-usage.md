# Impeccable 使用指南

> [Impeccable](https://impeccable.style/) 是 Claude Code 的前端设计增强插件，提供 16 个设计 slash 命令，覆盖审查、打磨、适配、增强等场景。

---

## 安装

```bash
/plugin marketplace add pbakaus/impeccable
```

## 首次配置

安装后先运行一次 `/teach-impeccable`，它会收集你的设计上下文（设计系统、色彩规范、字体偏好等）并保存到 AI 配置文件中，后续所有命令都会参考这份上下文。

```
/teach-impeccable
```

---

## 命令速查

### 1. 诊断类 Diagnostic

#### `/audit`
全面审查界面质量，生成详细报告和严重性评级。

```
# 审查当前项目的前端界面
/audit

# 典型输出：对齐问题(P2)、颜色对比度不足(P1)、间距不一致(P3) 等
```

**适用场景：** 代码写完后、PR 提交前、设计评审时。

#### `/critique`
从 UX 角度评估设计效果，给出可操作的反馈。

```
/critique
```

**与 `/audit` 的区别：** `/audit` 偏技术质量（对齐、间距、一致性），`/critique` 偏用户体验（信息层级、操作流程、认知负担）。

---

### 2. 质量类 Quality

#### `/polish`
发布前最终打磨 — 修复对齐、间距、一致性、细节问题。

```
# 最后一遍精修
/polish
```

**适用场景：** 功能开发完成、准备上线前的最后一步。

#### `/normalize`
统一设计，使其匹配你的设计系统，确保一致性。

```
/normalize
```

**适用场景：** 多人协作后风格不统一、引入第三方组件后需要统一视觉。

#### `/optimize`
优化性能 — 加载速度、渲染、动画、图片、包体积。

```
/optimize
```

**适用场景：** 页面加载慢、动画卡顿、Lighthouse 分数低。

#### `/harden`
增强界面健壮性 — 错误处理、国际化（i18n）、文本溢出处理。

```
/harden
```

**适用场景：** 准备国际化、需要处理边界情况（超长文本、网络错误、空数据）。

---

### 3. 强度调节 Intensity

#### `/bolder`
把保守、无聊的设计变得更有视觉冲击力和吸引力。

```
/bolder
```

**适用场景：** 设计太平淡、Landing Page 没有吸引力、需要更强视觉表达。

#### `/quieter`
降低过于激进的视觉设计，保持质量但更加克制。

```
/quieter
```

**适用场景：** 颜色太多太杂、动效过于花哨、视觉噪音太大。

---

### 4. 适配类 Adaptation

#### `/clarify`
改善不清晰的 UX 文案、错误提示、微文案、标签和说明。

```
/clarify
```

**适用场景：** 按钮文案含糊、错误提示不友好、表单标签不明确。

#### `/distill`
去除不必要的复杂性，提炼设计本质。

```
/distill
```

**适用场景：** 界面元素太多、功能堆砌、用户不知道该看哪里。

#### `/adapt`
适配不同屏幕尺寸、设备、上下文或平台。

```
/adapt
```

**适用场景：** 桌面端做完要适配移动端、需要支持暗色模式、跨平台适配。

---

### 5. 增强类 Enhancement

#### `/animate`
添加有意义的动画、微交互和动效。

```
/animate
```

**适用场景：** 页面转场生硬、按钮缺少反馈、列表加载没有过渡效果。

#### `/colorize`
为缺乏视觉吸引力的界面添加策略性配色。

```
/colorize
```

**适用场景：** 界面太素、灰度太多、需要用颜色引导用户注意力。

#### `/delight`
添加惊喜感和个性化触感，让界面令人难忘。

```
/delight
```

**适用场景：** 成功页面、空状态、加载等待、彩蛋效果。

---

### 6. 系统类 System

#### `/teach-impeccable`
一次性设置 — 收集你的设计上下文并保存到 AI 配置文件。

```
/teach-impeccable
```

**使用时机：** 首次安装后必须执行一次。设计规范变更后可重新执行。

#### `/extract`
提取并整合可复用的组件、设计 token 和模式到你的设计系统。

```
/extract
```

**适用场景：** 项目中有重复的设计模式需要抽象、建立/完善设计系统。

#### `/onboard`
设计或改善引导流程、空状态和首次使用体验。

```
/onboard
```

**适用场景：** 新功能需要引导、空状态页面设计、首次登录体验优化。

## 推荐工作流

### 日常开发

```
写代码 → /audit → 修复问题 → /polish → 提交
```

### 新功能上线

```
/teach-impeccable（首次）
     ↓
写前端代码
     ↓
/audit + /critique → 修复
     ↓
/adapt（响应式适配）
     ↓
/harden（健壮性）
     ↓
/animate（微交互）
     ↓
/polish（最终打磨）
     ↓
/optimize（性能优化）
     ↓
上线
```

### 设计调优

```
太无聊？ → /bolder → 过了？ → /quieter
太复杂？ → /distill
文案不好？ → /clarify
缺少色彩？ → /colorize
缺少动感？ → /animate
缺少惊喜？ → /delight
```

### 设计系统建设

```
/extract → 提取可复用模式
/normalize → 统一现有界面
/teach-impeccable → 更新设计上下文
```
